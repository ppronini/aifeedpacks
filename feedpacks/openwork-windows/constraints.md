# Constraints — жёсткие ограничения

## Установка

- ❌ **Не устанавливать Python из Microsoft Store** — это заглушка, `python.exe` открывает магазин
- ❌ **Не использовать PowerShell 7** — OpenWork использует PowerShell 5.1
- ❌ **Не использовать `&&`** — недоступен в PowerShell 5.1
- ❌ **Не менять директорию через `cd`** — агент должен использовать параметр `workdir`
- ✅ Использовать `cmd1; if ($?) { cmd2 }` для цепочек команд
- ✅ Использовать полные пути к exe-файлам

## Пути

- ✅ Проекты хранить на отдельном диске (не C:\)
- ✅ Все пути с пробелами — в двойных кавычках с `&`
- ✅ Для временных файлов — `%TEMP%\opencode\`
- ❌ Не полагаться на PATH без проверки

## Разрешения

- ❌ **Не игнорировать "permission denied"** — нужно добавить папку в Settings → Permissions
- ✅ После добавления папки — вернуться в чат и повторить команду
- ✅ Можно добавить несколько папок: workspace, XAMPP, temp

## Кодировка

- ❌ **Не работать с русским текстом без UTF-8** — будут кракозябры
- ✅ Выполнить `chcp 65001` перед работой
- ✅ Или включить UTF-8 в параметрах региона Windows

## Скрипты

- ❌ **Не запускать Python-скрипты без проверки ExecutionPolicy**
- ✅ `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`

## OpenWork специфика

- ❌ **Не путать OpenWork с OpenAI** — это разные продукты
- ✅ OpenWork = десктопное приложение-агент
- ✅ OpenAI = провайдер AI-моделей
