# AI FeedPacks

## Vision

AI FeedPacks is a public library of reusable Markdown bundles designed to be consumed by AI systems.

A FeedPack is not a prompt.

A FeedPack is a structured collection of Markdown files containing context, instructions, workflows, examples, constraints, and output requirements that help AI tools perform a specific task consistently and effectively.

The goal is simple:

**Feed better context. Get better output.**

## Core Concept

Modern AI tools perform significantly better when provided with structured context rather than isolated prompts.

AI FeedPacks provides ready-to-use bundles that users can download, copy, customize, and feed directly into:

* ChatGPT
* Claude
* Gemini
* Cursor
* Windsurf
* GitHub Copilot
* OpenWebUI
* Local LLMs
* AI Agents

Each FeedPack represents reusable expertise.

Examples:

* PHP Best Practices FeedPack
* Laravel Code Review FeedPack
* Startup Landing Page FeedPack
* SEO Audit FeedPack
* Product Requirements FeedPack
* Customer Interview Analysis FeedPack
* API Design FeedPack
* Bug Triage FeedPack

## Definition

### What is a FeedPack?

A FeedPack is a reusable bundle of Markdown files that gives an AI system everything it needs to complete a task.

A FeedPack may contain:

* Purpose
* Context
* Instructions
* Workflows
* Input requirements
* Output formats
* Examples
* Constraints
* Quality checklists
* References

## FeedPack Components

### README.md

Overview of the FeedPack.

Contains:

* Purpose
* Intended audience
* Use cases
* Installation / usage instructions

### instructions.md

Core AI instructions.

Defines:

* Role
* Behavior
* Rules
* Constraints
* Tone
* Reasoning expectations

### workflow.md

Step-by-step process.

Example:

1. Analyze input
2. Validate assumptions
3. Perform task
4. Generate output
5. Run quality checks

### inputs.md

Required information.

Example:

* Source code
* Business requirements
* User stories
* Existing documentation

### output-format.md

Defines expected output structure.

Examples:

* Markdown
* JSON
* Table
* Executive summary
* Technical specification

### examples.md

Examples of:

* Inputs
* Outputs
* Good responses
* Edge cases

### constraints.md

Hard rules.

Examples:

* Never invent facts
* Always cite assumptions
* Avoid marketing language
* Follow coding standards

### checklist.md

Final validation checklist.

Example:

* Requirements addressed
* Errors identified
* Format validated
* Recommendations prioritized
